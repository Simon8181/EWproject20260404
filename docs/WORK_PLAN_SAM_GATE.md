# 工作目标与 Sam 汇报门槛（订单账面 + 网页）

**约定**：本计划以「**达到下方目标后再找 Sam**」为统一节奏；你（业务）与开发分工见各阶段。

---

## 一、找 Sam 时要达成的目标（门槛）

同时具备下面 **3 条**，即视为可以约 Sam 做正式汇报与拍板（上线范围、权限、资源）：

1. **订单账面可汇总**  
   对「我们已接入的订单数据」（至少 **下单 / order** 这条线），在网页上能**按单**看到与 `COMPANY_RULES.md` 对齐的简化关系：  
   **账面差 ≈ 客户报价（Sheet **P 列** `quote_customer`）− 司机报价（**U 列** `quote_driver`）− 其他费用（`other_fee`，不在 Sheet，**网页追加录入**，表 `order_fee_addons`）**  
   - 说明：最终**会计口径**以财务为准；系统先做到**可解释、可导出、与 Sheet 可对账**，再迭代精度。

2. **网页端可用、可演示**  
   在**手机 + 电脑**上能完成：**看列表 → 点进一单 → 看关键字段 + 账面摘要 + 地图**；风格**参考 DAT**（见下「设计参考」），不追求与 DAT 像素级一致。

3. **你方补充材料已落地**  
   已确认：**P=客户报价、U=司机报价**；**其他费用**走网页追加 + 表 `order_fee_addons`。若再增加口径（杂费细分、币种），以书面补充为准。

---

## 二、DAT 风格参考（网页，非商标）

参考 **DAT 类 load board** 的常见体验（高对比、**橙色强调**、深色或浅色底上的清晰卡片/列表、里程与路线信息突出），用于：

- 主色倾向：**橙** `#FF6B00`～`#E85D04` 作 CTA/高亮；背景深灰 `#0f1117`～`#1a1d26` 或浅灰底 + 深色字（二选一，实现时统一一套 token）。
- 信息层级：**一行一单摘要**（EW 号、状态、Mi、账面差颜色标识）→ 详情页沿用现有订单卡结构并加强「账面」区块。
- 交互：大点击区域、适合司机/Broker 单手操作（与现有 `order_view` 移动端方向一致）。

实现时以 **CSS 变量**集中管理（`--dat-orange`, `--bg`, `--text`），便于你一句话改品牌色。

## 三、分阶段工作计划（你配合项）

### 阶段 0：对齐（你主导，1 次内完成）

| 事项 | 产出 |
|------|------|
| 确认「一单」在 Sheet 里的**唯一键**（已有 `ew_quote_no`） | 书面确认 |
| **客户报价 / 司机报价** | **P → `quote_customer`，U → `quote_driver`**（已写入 `EW_ORDER_RULES.yaml`） |
| **其他费用** | **网页追加**，表 **`order_fee_addons`**（待接 API / 录入 UI） |
| 货值等其它列 | 仍以 `O` 等现有映射为准；若再扩列，书面补充 |

**开发**：`EW_ORDER_RULES.yaml` + `db/schema_order.sql`（`ew_orders`）已含 P/U 与 `order_fee_addons`；下一步为追加费用 API 与账面公式。

---

### 阶段 1：数据层 — 账面字段进库或可计算

| 事项 | 产出 |
|------|------|
| 映射 + 同步 | 关键金额字段进入 Postgres（或只读视图公式在应用层算） |
| 单笔计算 | `margin ≈ quote_customer − quote_driver − other_fee`（`other_fee` 来自 `order_fee_addons` 汇总，实现时定） |
| 校验 | 与 Sheet 抽几单**人工对账**通过 |

**你配合**：提供 3～5 单样例「预期账面差」，开发核对系统输出。

---

### 阶段 2：网页 — 列表 + 详情 + DAT 风

| 事项 | 产出 |
|------|------|
| 新路由 | 如 `/orders` 或 `/f/summary/orders`（实现时与现有 `ew_service` 统一） |
| 列表页 | 每行：EW、Mi、账面差（颜色）、状态摘要；筛选/排序（至少按 EW、时间） |
| 详情 | 现有 order 卡 + **账面摘要条**（收入/成本/杂费/差） |
| 样式 | 应用 DAT 参考 token，与主页、通用表分页风格一致 |

**你配合**：试用手机端，反馈「一眼要看哪三个数」——可调整列表列顺序。

---

**达到阶段 1+2 且阶段 0 对齐无歧义** → **约 Sam**。

## 四、风险与边界（汇报时诚实说）

- Sheet 若**未录**承运成本，账面差会**不全**——需约定「先填后算」或「仅显示客户侧+待补成本」。
- 前四 Sheet **编辑权限**仍属公司制度 + Sam，与本次「读数+展示」可分阶段上。
- **会计/税务**最终口径以财务为准，系统标「内部管理看板」。
